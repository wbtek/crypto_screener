Expand description

Represents the main application model, managing the application’s core state.

Model includes fields for application data, error messages, modal visibility, sorting options, and cell selections. It implements the Yew Component trait, handling updates and rendering based on incoming messages (Msg).

pub fn sort_order(&self, column: &str) -> SortOrder

Determines the current sort order for a specified column.

This method checks if the specified column is the one currently being sorted in the model. If so, it returns the appropriate sort order (Ascending or Descending) based on the sort_asc field. If the column is not currently sorted, it returns SortOrder::None.

§Parameters
  • column: The name of the column to check for the current sort order.
§Returns

Returns a SortOrder indicating the current sorting state:

  • SortOrder::Ascending if column matches the currently sorted column and sort_asc is true.
  • SortOrder::Descending if column matches the currently sorted column and sort_asc is false.
  • SortOrder::None if column does not match the currently sorted column.
§Example
let model = Model {
    sort_by: Some("price_usd".to_string()),
    sort_asc: true,
    ..Default::default()
};
assert_eq!(model.sort_order("price_usd"), SortOrder::Ascending);
assert_eq!(model.sort_order("symbol"), SortOrder::None);
